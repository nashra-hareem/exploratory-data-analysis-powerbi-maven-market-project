# Exploratory Data Analysis using Power BI

This project started as a structured dataset, but the goal was not just to build visuals. Early on, it became clear that surface level trends were easy to identify, but they did not explain much on their own.

After cleaning and transforming the data in Power Query, I began with basic breakdowns to understand direction. However, those initial views felt incomplete. The numbers made sense, but they did not answer why.

So the approach shifted. Instead of focusing on what the data shows, I started questioning what it might be hiding.

I used DAX to create measures that allowed me to slice the data across different dimensions and compare patterns more closely. Some trends stayed consistent, while others changed once broken down further. In several cases, smaller segments and outliers explained more than overall averages.

The process became less about building dashboards and more about understanding what was actually driving the numbers. This meant moving past what was immediately visible and spending time with inconsistencies, edge cases, and variations that did not fit neatly into the bigger picture.

## Approach

The analysis followed a simple progression:

1. Clean and structure the data so it can actually be trusted
2. Understand high level trends without overinterpreting them
3. Break the data into segments to test whether those trends hold
4. Use DAX to explore relationships and compare different views
5. Focus on what changes when the perspective changes
